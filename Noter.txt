

Windows ville ikke tage kontakt med API'en fra hjemmesiden. (Pga. optaget port 8080 på windows)

Hjemmesiden gav heller ikke samme fejlmelding uden API backend på Windows som på linux.
På Windows gav hjemmesiden ingen fejlmelding på hjemmesiden, Hvor linux skriver: Kunne ikke kontakte API: Failed to fetch
Grund: Ved optaget port 8080, forsøger frontend at kontakte en forkert app som backend, og når den forkerte app på 8080 aldrig svare kommer den aldrig videre.


Check used ports (for 8080) on windows: 
netstat -ano | findstr :8080

Find processen med PID 1234 på windows:
tasklist /fi "pid eq 1234"

Kill process with PID in powershell as administrator:
Stop-Process -Id 1234 -Force
Stop-Process -Id 4836 -Force
